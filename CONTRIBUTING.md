# Contributing

This repository is meant to stay useful as a living research guide, not just as a static report. Small corrections, vendor updates, lifecycle changes, and field notes are all valuable.

## Update Workflow

1. Edit the relevant Markdown file.
2. Keep recommendations grounded in dated evidence.
3. Mark uncertain claims with `[unverified]`, `[supplier claim]`, or `[inferred]`.
4. Run `make pdf` to rebuild the combined PDF.
5. Commit the Markdown changes and the regenerated PDF together.

## Source Quality

Prefer primary sources whenever possible:

- vendor datasheets and lifecycle statements
- official support documents
- kernel or project documentation
- teardown reports with photos
- dated forum reports when vendor documentation is missing

Avoid turning one-off product listings into firm recommendations unless a supplier has confirmed the relevant details in writing.
